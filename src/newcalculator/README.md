도전 과제의 경우엔 처음에 굉장히 막막했다.

처음엔 enum 타입 변환으로 인하여 static 변수를 선언해주는 char operator 부분에서 그랬고 다음엔 throw 문에 대한 개념이 정확하게 잡혀 있지 않아 return 0;로 반환하려다 계속 오류를 뿜는 현상을 볼 수 있었다.

제네릭의 경우는 제네릭 확장을 응용할 줄 몰라서 <T>라고만 해둔 후에 doubleValue()를 사용하여 한참 동안 헤맸다. 결국 강의 복습과 검색을 통해 <T extends 상위타입>을 통해 응용이 가능하다는 것을 알았고 <T extends Number>를 통하여 doubleValue()를 정상적으로 사용할 수 있었다.

하지만 람다 스트림을 이용한 구문은 public으로 시작한다는 것은 알았지만 도무지 어떻게 선언해줘야 하는지 감이 잡히지를 않았다.

분명 App.java에서 주석 처리된 문장을 이용하고 ArithmeticCalculator.java에서 stream(), 값을 걸러주는 filter(), map()을 사용하면 될 것이라고 예상은 했지만 아래 코드에서도 double value에서 스캐너 입력 값을 받아준 이후의 구문이 떠오르지 않았다.

강의를 다시 들어보고 검색을 열심히 해본 결과 조금 더 이해가 되었고 다시 람다 스트림을 구현해봤다.

ArithmeticCalculator 클래스 내 스트림에서 List가 사용되고 다른 클래스에서도 스트림을 불러올 것이므로 public을 사용하여 List<Double> big (double value)로 big이란 이름의 리스트와 double 형태의 value를 만들어줬다.

그리고 List<Double> box를 만들어 저장된 값을 스트림으로 변환시켜 주었고 .filter를 통하여  num > value인 숫자를 걸러내 List로 수집했다.

App 클래스 내에선 scanner를 이용하여 실수를 입력 받을 수 있게 해줬고 처음에 scanner.nextLine();을 빼먹어 자꾸 오류가 나타났었다. 다시 한번 버퍼 타입 초기화의 중요성을 깨닫는 순간이었다.

그 다음 List<double> box 구문을 가져와 스트림을 이용할 수 있게 해줘 람다 스트림 구현을 마무리했다.

마지막으로 ArithmeticCalculator 클래스 내 switch 문에서 DIV와 default의 경우 처음에 return 0;로 오류를 표현했는데 이렇게 표현하면 그저 반환 값을 0으로 표현하게 되므로 오류를 제대로 표현할 수 없어 throw를 활용하여 정상적으로 오류를 표현할 수 있도록 수정해주었다.
